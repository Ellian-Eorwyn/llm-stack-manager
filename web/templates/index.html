How To Use This Page

Each section saves independently. Primary Backend and Secondary Backend each have their own proxy and model-routed aggregate endpoint.

{% for section in config_sections.keys() %} {% endfor %}

{% if section == 'Primary Backend' %} Configure the primary llama.cpp backend. It binds to the primary internal backend port used by Primary Proxy. {% elif section == 'Secondary Backend' %} Configure the secondary llama.cpp backend. It binds to its own internal backend port and is served by Secondary Proxy. {% elif section == 'Thinking Endpoint' %} Request-time defaults for the thinking proxy port. These overrides are injected by the proxy without reloading the shared backend model. {% elif section == 'Instruct Endpoint' %} Request-time defaults for the non-thinking instruct proxy port. Use this pane to keep chat-style behavior separate from the thinking and coding routes. {% elif section == 'Coding Endpoint' %} Request-time defaults for the coding proxy port. These settings let coding clients see a different exposed model identity and sampling profile without touching the loaded backend. {% elif section == 'Task Model' %} Tune the smaller task model used for fast utility work and background tasks. {% elif section == 'OCR' %} Configure the standalone GLM-OCR multimodal backend and its default extraction behavior. {% elif section == 'Chat Templates' %} Create and edit reusable llama.cpp Jinja templates, then select them in Primary Backend, Secondary Backend, or Task Model. {% else %} Update this section and save it independently. Changes apply to the environment file used by the stack manager. {% endif %}
